# Changelog

Все значимые изменения проекта документируются здесь.
Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/).
Проект пока без выпущенных версий.

## [Unreleased]

### Добавлено
- Сканирование дисков с подсчётом фактически занимаемого места на томе
  (округление до кластера, сжатие NTFS, резидентные в MFT файлы).
- Дедупликация жёстких ссылок при подсчёте размеров.
- Устойчивый обход: пропуск каталогов при отказе доступа (`SafeEnumeration`),
  пометка недоступных каталогов (`IsAccessDenied`).
- `DriveWatcher` — слежение за изменениями файловой системы.
- WPF-интерфейс: список томов с метками, сводка по диску, дерево каталогов
  с колонками «На диске»/«Размер», круговая диаграмма текущего уровня с
  навигацией двойным кликом, подвижная граница между деревом и диаграммой.
- Живые обновления: авто-перескан по простою (с тумблером).
- Окно прогресса с отменой (появляется, если скан дольше 3 секунд).
- Сегмент «Прочее/системное» для защищённых областей и метаданных ФС.
- Блокировка навигации в точки соединения и недоступные области с подсветкой.
- Юнит-тесты ядра и прикладной логики (xUnit), README, CHANGELOG.

### Изменено
- MVVM переведён на отдельный пакет `Ti-Soft.MvvmUtilites` (вместо копии кода).
- Проекты UI и тесты переведены на .NET 10.

### Исправлено
- Неверный подсчёт размера файлов больше 4 ГБ (потеря старшего слова при сдвиге).
- Упаковка атрибутов в `BitVector32` (использовалась маска вместо индекса бита).
- `InvalidCastException` в определении размера кластера (`BlockSize` в WMI — `uint64`).
- Кнопка «Сканировать» не запускала скан после смены диска.
- «Змейка» в дереве: колонки размеров не выравнивались под заголовками.

### Удалено
- Проект `Permissions` (`SeBackupPrivilege`) — для подсчёта объёма не требуется.
